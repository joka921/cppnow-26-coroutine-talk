Coroutine Anatomy

Two halves

  • Ramp — allocates frame, constructs promise, calls get_return_object, awaits initial_suspend
  • Actor — switch/goto state machine, executes body between suspension points

Frame lives on the heap • Promise object inside the frame • Each co_yield/co_await = suspension point


// C++20 iota generator
generator<int> iota(int start, int end) {
  while (start < end) {
    co_yield start++;
  }
}

// Usage
for (int v : iota(0, 5)) {
  std::cout << v << "\n";
}

Promise Protocol Recap

promise_type

  • get_return_object()
  • initial_suspend()
  • final_suspend()
  • yield_value(expr)
  • return_void() / return_value(v)
  • unhandled_exception()

Awaitable

  • await_ready() → bool
  • await_suspend(handle) → void / bool / handle
  • await_resume() → result

coroutine_handle

  • resume(), destroy(), done()
  • from_promise(p), promise()

// DESTROY (cleanup)

Building the Infrastructure

Handles, frames, storage, macros

HandleFrame

Type-erased base — two function pointers


// A type-erased "base" for a coroutine frame.
// resumeFunc returns a handle for symmetric transfer trampolining.
struct HandleFrame {
  stackless_coroutine_handle<void> (*resumeFunc)(void*);
  void (*destroyFunc)(void*);
};
  • resumeFunc returns a handle → enables symmetric transfer
  • destroyFunc cleans up the frame
  • Both take void* — type erasure via the frame pointer

stackless_coroutine_handle<void>


namespace detail {
struct stackless_handle_base {
  HandleFrame* ptr;

  stackless_handle_base() noexcept : ptr(nullptr) {}
  stackless_handle_base(HandleFrame* ptr) noexcept : ptr(ptr) {}

  void resume();  // trampoline — defined later
  void destroy() { ptr->destroyFunc(reinterpret_cast<void*>(ptr)); }
  operator bool() const { return static_cast<bool>(ptr); }
  bool done() const { return ptr->resumeFunc == nullptr; }
};
}

template <>
struct stackless_coroutine_handle<void>
    : public detail::stackless_handle_base {
  using stackless_handle_base::stackless_handle_base;
};

stackless_coroutine_handle<Promise>


template <typename Promise>
struct stackless_coroutine_handle
    : public detail::stackless_handle_base {
  using stackless_handle_base::stackless_handle_base;

  // Promise lives right after HandleFrame in the coroutine frame
  static constexpr size_t promise_offset =
      (sizeof(HandleFrame) + alignof(Promise) - 1)
      & ~(alignof(Promise) - 1);

  static stackless_coroutine_handle from_promise(Promise& p) {
    auto* ptr = reinterpret_cast<HandleFrame*>(
        reinterpret_cast<char*>(&p) - promise_offset);
    return stackless_coroutine_handle{ptr};
  }

  Promise& promise() {
    return *reinterpret_cast<Promise*>(
        reinterpret_cast<char*>(ptr) + promise_offset);
  }
};

Trampoline (Symmetric Transfer)


// After stackless_coroutine_handle<void> is complete:
inline void detail::stackless_handle_base::resume() {
  stackless_coroutine_handle<void> next =
      ptr->resumeFunc(reinterpret_cast<void*>(ptr));
  while (next.ptr) {
    next = next.ptr->resumeFunc(
        reinterpret_cast<void*>(next.ptr));
  }
}
  • resumeFunc returns the next handle to resume
  • Loop replaces recursive .resume() calls → flat stack
  • Null handle = "no more transfers, stop"

SuspendAlways / SuspendNever


struct SuspendAlways {
  static constexpr bool await_ready() noexcept { return false; }

  static constexpr void await_suspend(Handle) noexcept {}

  static constexpr void await_resume() noexcept {}

struct SuspendNever {
  static constexpr bool await_ready() noexcept { return true; }

  static constexpr void await_suspend(Handle) noexcept {}

coro_storage

Storing local variables across suspension points


template <typename Ref, bool isOwning>
struct coro_storage {
  // Owning: stores decay_t<Ref> by value
  // Non-owning: stores a pointer
  using Storage = std::conditional_t<isOwning,
      std::decay_t<Ref>,
      std::add_pointer_t<std::decay_t<Ref>>>;

  alignas(Storage) char buffer[sizeof(Storage)];

  struct Proxy { Ref ref_; };

  Proxy get() noexcept {
    Storage& s = *std::launder(reinterpret_cast<Storage*>(buffer));
    if constexpr (isOwning) return s;
    else return *s;
  }

  void destroy() noexcept(...) {
    std::launder(reinterpret_cast<Storage*>(buffer))->~Storage();
  }
};

CRTP Base Layout


template <typename Derived, typename PromiseType, bool isNoexcept>
struct stackless_coro_crtp {
  // Must match stackless_coroutine_handle layout:
  // promise_ directly after frame_
  HandleFrame frame_;
  PromiseType promise_;

  size_t suspendIdx_ = 0;
  size_t currentTryBlock_ = CO_NO_TRY_BLOCK;  // exception dispatch

  // Awaiters for initial_suspend() and final_suspend()
  coro_storage<decltype(promise_.initial_suspend())&, true>
      initial_awaiter_;
  coro_storage<decltype(promise_.final_suspend())&, true>
      final_awaiter_;

  // Constructor: wire up function pointers
  stackless_coro_crtp() {
    frame_.resumeFunc  = &stackless_coro_crtp::resume;
    frame_.destroyFunc = &stackless_coro_crtp::destroy;
  }

  // Derived must implement:
  //   doStepImpl()            — the actor state machine
  //   destroySuspendedCoro()  — cleanup at each suspend point
};

ramp() — The Coroutine Entry Point


template <typename... CoroArgs>
static auto ramp(CoroArgs&&... coroArgs) {
  // 1. Allocate the frame
  void* coroMem = promise_allocate<PromiseType>(
      sizeof(Derived), std::forward<CoroArgs>(coroArgs)...);

  // 2. Placement-new the Derived frame
  auto* frame = new(coroMem) Derived{
      std::forward<CoroArgs>(coroArgs)...};

  // 3. get_return_object (stored on caller's stack)
  auto ret = frame->promise_.get_return_object();

  // 4. co_await initial_suspend
  CO_STORAGE_CONSTRUCT(frame->initial_awaiter_,
      (frame->promise_.initial_suspend()));
  auto handle = frame->getHandle();
  CO_AWAIT_IMPL_IMPL(frame->initial_awaiter_.get().ref_,
                      handle, ret);

  // 5. If initial_suspend didn't suspend, resume immediately
  handle.resume();
  return ret;
}

doStep() — The Actor Function


stackless_coroutine_handle<void> doStep() noexcept(isNoexcept) {
  if constexpr (isNoexcept) {
    // No exception handling needed
    return derived().doStepImpl();
  } else {
    auto& d = derived();
    try {
      return d.doStepImpl();
    } catch (...) {
      auto [handle, frame_destroyed] = d.handleException();
      if (frame_destroyed) return handle;

      // Exception was caught, re-enter state machine
      if (!d.done()) {
        return d.doStep();  // recursive call for catch dispatch
      }
      return handle;
    }
  }
}

The try/catch wrapper enables coroutine-level exception dispatch

destroy()


void destroy() {
  if (!derived().done()) {
    // Still suspended mid-body: clean up locals
    derived().destroySuspendedCoro(derived().suspendIdx_);
  } else {
    // At final suspension point: only final_awaiter_ remains
    derived().final_awaiter_.get().ref_.await_resume();
    derived().final_awaiter_.destroy();
  }
  deleteFrame();  // destructor + operator delete
}
  • destroySuspendedCoro(idx) knows exactly which locals are alive at each suspension point
  • Mirror of the compiler's "cleanup based on suspend point" logic

Macro Reference


// Access stored value
#define CO_GET(mem)  this->mem.get().ref_

// Construct into coro_storage
#define CO_INIT(mem, ...)  CO_STORAGE_CONSTRUCT(this->mem, __VA_ARGS__)

// co_yield value
#define CO_YIELD(index, awaiterMem, value)
  // yield_value → construct awaiter → set suspendIdx
  // → await → label → await_resume → destroy awaiter

// co_await expr
#define CO_AWAIT(index, awaiterMem, expr, ...)
  // get_awaiter → construct awaiter → set suspendIdx
  // → await → label → result = await_resume

// co_return
#define CO_RETURN_VOID(index, finalAwaiterMem)
  // return_void → destroySuspendedCoro → setDone
  // → final_suspend → await final awaiter

#define CO_RETURN_VALUE(index, finalAwaiterMem, value)
  // return_value → ... same as above

iota — doStepImpl() 


coroutine_handle<void> doStepImpl() {
  // Jump table: resume at the right suspension point
  switch (suspendIdx_) {
    case 0: break;          // first entry (after initial_suspend)
    case 1: goto label_1;   // resume after co_yield
  }

  // After initial_suspend:
  CO_GET(initial_awaiter_).await_resume();
  initial_awaiter_.destroy();

  while (start_ < end_) {
    CO_YIELD(1, initial_awaiter_, (start_));
    ++start_;
  }

  // implicit co_return 
  CO_RETURN_VOID(2, final_awaiter_);
}

iota — destroySuspendedCoro()


void destroySuspendedCoro() {
  switch (suspendIdx_) {
    case 0:
      // Suspended at initial_suspend
      initial_awaiter_.destroy();
      return;
    case 1:
      // Suspended at co_yield (mid-loop)
      initial_awaiter_.destroy();
      return;
    case 2:
      // At co_return — no local variables to destroy
      return;
  }
}
  • Each case corresponds to a suspension point
  • Lists exactly which objects are alive and need destruction
  • The compiler generates equivalent tables internally

Example (b)

Local Variables Across Suspension

string_prepend — C++20


generator<std::string> string_prepend(
        auto&& rangeOfStrings, std::string prefix) {
  auto it = rangeOfStrings.begin();
  auto end = rangeOfStrings.end();
  for (; it != end; ++it) {
    co_yield (prefix + *it);
  }
}
  • Iterator variables live across suspension points
  • prefix + s creates a temporary that must survive the yield
  • These locals need explicit lifetime management in the frame

string_prepend — Frame Layout


struct CoroFrame
    : coro_crtp<CoroFrame, promise_type> {
  RangeOfStrings&& rangeOfStrings_;
  std::string prefix_;

  // Local variables stored as coro_storage:
  using It  = std::remove_cvref_t<decltype(rangeOfStrings_.begin())>;
  using End = std::remove_cvref_t<decltype(rangeOfStrings_.end()));>;

  coro_storage<It&, true> it_;    // iterator
  coro_storage<const End&, true> end_; // sentinel
  coro_storage<std::string&&, true> tmp_;   // yield temp

  CoroFrame(RangeOfStrings&& ros, std::string pref)
      : rangeOfStrings_(std::forward<RangeOfStrings>(ros)),
        prefix_(std::move(pref)) {}
};

What does the type of `tmp_` depend on?

string_prepend — doStepImpl()


stackless_coroutine_handle<void> doStepImpl() {
  switch (suspendIdx_) {
    case 0: break;
    case 1: goto label_1;
  }

  CO_GET(initial_awaiter_).await_resume();
  initial_awaiter_.destroy();

  CO_INIT(it_,  (rangeOfStrings_.begin()));
  CO_INIT(end_, (rangeOfStrings_.end()));

  for (; CO_GET(it_) != CO_GET(end_); ++CO_GET(it_)) {
    CO_INIT(tmp_, (prefix_ + *CO_GET(it_)));  // construct temp
    CO_YIELD(1, initial_awaiter_, CO_GET(tmp_));
    tmp_.destroy();                            // destroy after resume
  }

  CO_RETURN_VOID(2, final_awaiter_);
}

string_prepend — destroySuspendedCoro()


void destroySuspendedCoro() {
  switch (suspendIdx_) {
    case 0:
      // At initial_suspend — only awaiter alive
      initial_awaiter_.destroy();
      return;
    case 1:
      // Mid-loop, at co_yield — tmp_ is alive!
      initial_awaiter_.destroy();
      tmp_.destroy(); 
    case 2: [[fallthrough]]
      end_.destroy();
      it_.destroy();
      return;
  }
}

coro_storage Design

  • Owning (isOwning=true): stores the object by value in the buffer
    • Used for temporaries, rvalue refs, objects that must outlive suspension
  • Non-owning (isOwning=false): stores a pointer
    • Used for lvalue references — the referent outlives the coroutine
  • Proxy pattern: storage.get().ref_ gives the correct reference type
    • Rvalue refs stay rvalue refs (no accidental std::move through a getter)
  • Placement new + explicit destroy: manual lifetime, no default construction needed

Side-by-Side Comparison

     : rangeOfStrings) {

}

C++17 (hand-lowered)

    (rangeOfStrings_.begin()));

     ++CO_GET(it_)) {
  CO_INIT(tmp_,

            CO_GET(tmp_));

The structure is preserved. The ceremony is the lifetime management.

Example (c)

Task with Symmetric Transfer

What Is Symmetric Transfer?

The problem

  • Task A co_awaits Task B
  • Task B co_awaits Task C
  • Each .resume() adds a stack frame
  • Deep chains → stack overflow

The solution

  • await_suspend returns a handle
  • Caller resumes that handle instead of the current one
  • Trampoline loop — flat stack
  • No recursion, O(1) stack depth

task_promise


template <typename T, template <typename...> typename TaskHandle>
class task_promise {
public:
  using handle_type = TaskHandle<task_promise>;

  ::task<T, TaskHandle> get_return_object() noexcept;

  // Lazy: always suspend at start
  constexpr SuspendAlways initial_suspend() const noexcept { return {}; }

  // Symmetric transfer: return to caller on completion
  task_final_awaiter<TaskHandle> final_suspend() const noexcept
  { return {}; }

  void return_value(T value) { value_ = std::move(value); }
  void unhandled_exception() {
    exception_ = std::current_exception();
  }

  TaskHandle<void> continuation_;  // who to resume when done

private:
  std::optional<T> value_;
  std::exception_ptr exception_;
};

task_final_awaiter


template <template <typename...> typename TaskHandle>
struct task_final_awaiter {
  static constexpr bool await_ready() noexcept { return false; }

  template <typename Promise>
  TaskHandle<void> await_suspend(TaskHandle<Promise> h) noexcept {
    if (h.promise().continuation_) {
      return h.promise().continuation_;   // resume caller
    }
    return noop_coroutine();              // top-level: stop
  }

  static constexpr void await_resume() noexcept {}
};
  • await_suspend returns a handle → symmetric transfer
  • If there's a continuation (caller), resume it
  • Otherwise, return noop_coroutine() to stop the trampoline

task_awaiter


template <typename T, template <typename...> typename TaskHandle>
class task_awaiter {
  using handle_type = TaskHandle<task_promise<T, TaskHandle>>;

public:
  explicit task_awaiter(handle_type coro) noexcept : coro_(coro) {}

  bool await_ready() const noexcept { return coro_.done(); }

  // Symmetric transfer: store caller, return inner task's handle
  template <typename CallerPromise>
  handle_type await_suspend(
      TaskHandle<CallerPromise> caller) noexcept {
    coro_.promise().continuation_ = caller;  // remember caller
    return coro_;                             // resume inner task
  }

  decltype(auto) await_resume() {
    return coro_.promise().result();
  }

private:
  handle_type coro_;
};

task<size_t> add_values(size_t a, size_t b) {

add_values — Frame Layout


struct CoroFrame
    : stackless_coro_crtp<CoroFrame, promise_type, false> {
  size_t a_, b_;
  size_t res_, va_, vb_;

  // The inner task and its awaiter need to live across suspension

      task_storage_;

  • awaiter_storage_: the task_awaiter for symmetric transfer

add_values — doStepImpl()


coroutine_handle<void> doStepImpl() {
  switch (this->suspendIdx_) {
    case 0: break;
    case 1: goto label_1;
    case 2: goto label_2;
  }

  CO_GET(initial_awaiter_).await_resume();
  initial_awaiter_.destroy();

  res_ = 0;
  while (a_ < b_) {
    // size_t va = co_await compute_value(a_);
    CO_INIT(task_storage_, (compute_value(a_)));
    CO_AWAIT(1, awaiter_storage_, CO_GET(task_storage_), va_ =);
    task_storage_.destroy();

    // size_t vb = co_await compute_value(b_);
    CO_INIT(task_storage_, (compute_value(b_)));
    CO_AWAIT(2, awaiter_storage_, CO_GET(task_storage_), vb_ =);
    task_storage_.destroy();

    res_ += va_ + vb_;
    ++a_;  --b_;
  }

  co_RETURN_VALUE(3, final_awaiter_, res_);
}

Symmetric Transfer Flow

  caller
    │
    ▼ .start()
┌──────────┐
│add_values│──CO_AWAIT──▶ task_awaiter::await_suspend
│  (a, b)  │             stores caller as continuation
└──────────┘             returns compute_value's handle
                              │
                              ▼ trampoline resumes
                        ┌──────────────┐
                        │compute_value │
                        │   (a)        │
                        └──────────────┘
                              │ co_return x*2
                              ▼
                        task_final_awaiter::await_suspend
                        returns continuation (= add_values)
                              │
                              ▼ trampoline resumes
                        ┌──────────┐
                        │add_values│  ← resumes at label_1
                        └──────────┘

Flat trampoline, no stack growth.

Example (d)

Exception Handling

"The hardest part"

throwing_parse_ints — C++20


generator<int> throwing_parse_ints(
    auto&& strings, bool catch_errors) {
  for (const auto& s : strings) {
    try {
      co_yield std::stoi(s);
    } catch (const std::invalid_argument&) {
      if (!catch_errors) throw;
      continue;
    } catch (const std::out_of_range&) {
      if (!catch_errors) throw;
      break;
    }
  }
}

co_yield inside a try-block, continue and break inside catch handlers.

Two Challenges

  1. Resuming into a try block
    • goto may not jump into a try block
    • Not even with asm goto on Itanium ABI
    • Solution: subdispatch in every try-block that contains suspension points
  2. Stack Unwinding
    • Exception can occur at any point — which locals are alive?
    • Solution: At runtime keep track of all the in-scope locals
    • Unfortunately, not zero overhead

Frame — Construction Tracking


struct CoroFrame: coro_crtp<CoroFrame, promise_type> {
  RangeOfStrings&& strings_;
  bool catch_errors_;

  using It  = std::remove_cvref_t<decltype(rangeOfStrings_.begin())>;
  using End = std::remove_cvref_t<decltype(rangeOfStrings_.end()));>;

  coro_storage<It&, true> it_;    // iterator
  coro_storage<const End&, true> end_; // sentinel
  coro_storage<int&, true>                    parsed_;

  // Track which locals have been constructed
  struct {
    bool initial_awaiter_ = false;
    bool it_ = false;
    bool end_ = false;
    bool parsed_ = false;
  } __constructed ;
};

Note: We coulD OMIt the tracking for trivially destructible types

doStepImpl() — Try Block + Labels


stackless_coroutine_handle<void> doStepImpl() {
  switch (this->suspendIdx_) {
    case 0:  break;
    case 1:  goto label_1;       // resume at co_yield
    case 12: goto try_end;       // resume after caught exception
    case 20: goto for_continue;  // continue from catch
    case 21: goto for_break;     // break from catch
  }

  CO_GET(initial_awaiter_).await_resume();
  DESTROY_UNCONDITIONALLY(initial_awaiter_);
  CO_INIT_EX(it_,  (this->strings_.begin()));
  CO_INIT_EX(end_, (this->strings_.end()));

  for (; CO_GET(it_) != CO_GET(end_); ++CO_GET(it_)) {
    this->currentTryBlock_ = 0;                      // enter try
    CO_INIT_EX(parsed_, (std::stoi(*CO_GET(it_))));
    CO_YIELD(1, initial_awaiter_, CO_GET(parsed_));
    DESTROY_UNCONDITIONALLY(parsed_);
    this->currentTryBlock_ = CoroFrameBase::CO_NO_TRY_BLOCK; // exit try
  try_end:
  for_continue:
    ;
  }
for_break:
  DESTROY_UNCONDITIONALLY(end_);
  DESTROY_UNCONDITIONALLY(it_);
  CO_RETURN_VOID(2, final_awaiter_);
}

dispatchExceptionHandling()


ExceptionResult dispatchExceptionHandling() {
  switch (this->currentTryBlock_) {
    case CoroFrameBase::CO_NO_TRY_BLOCK: {
      // Outside any try-block → unhandled
      DESTROY_IF_CONSTRUCTED(end_);
      DESTROY_IF_CONSTRUCTED(it_);
      DESTROY_IF_CONSTRUCTED(initial_awaiter_);
      return this->unhandled_exception();
    }
    case 0:
      return handleCatchClause_0();
    default:
      __builtin_unreachable();
  }
}
  • Called from the catch(...) in doStep()
  • Routes to the right handler based on currentTryBlock_
  • Outside any try-block: destroy what's alive, call unhandled_exception()

handleCatchClause_0()


ExceptionResult handleCatchClause_0() {
  this->currentTryBlock_ = CoroFrameBase::CO_NO_TRY_BLOCK;
  try {
    DESTROY_IF_CONSTRUCTED(parsed_);
    this->suspendIdx_ = 12;    // default: resume after try-catch
    try {
      throw;                    // rethrow to match catch clauses
    } catch (const std::invalid_argument&) {
      if (!catch_errors_) throw;
      this->suspendIdx_ = 20;  // continue → loop end
    } catch (const std::out_of_range&) {
      if (!catch_errors_) throw;
      this->suspendIdx_ = 21;  // break → after loop
    }
    return {};  // exception handled, resume with new suspendIdx_
  } catch (...) {
    // No catch matched, or rethrew → propagate up
    return dispatchExceptionHandling();
  }
}

Why Rethrow?

  • A coroutine cannot suspend inside a catch handler
    • The exception object would be destroyed on stack unwind
  • Solution: catch clause sets suspendIdx_ and returns
  • doStep() sees the exception was handled, calls doStep() again
  • doStepImpl() resumes at the new suspendIdx_ — outside any catch

// In doStep():
catch (...) {
  auto [handle, frame_destroyed] = d.handleException();
  if (!d.done()) {
    return d.doStep();  // re-enter with new suspendIdx_
  }
}

Stackful Alternative

No Heap, No Indirection

Until now, we have manually implemented coroutines with the exact semantics specified by C++20 (minus some inaccuracies and bugs introduced by myself of course). But since we have done everything ourselves, we also can explore different design choices, in particular stackful, strongly typed coroutines. Gladly, we can keep our tediously rewritten coroutine frame state machines, and only have to adapt the generic infrastructure like coroutine handles.

The Problem with Stackless

  • Heap allocation per coroutine frame
    • HALO helpful, but unreliable
  • Type erasure + function pointer indirection hinder optimizations
  • Unacceptable overhead for tight loops.

Can we do better when the coroutine type is known at the call site?

Before diving into alternative designs, let's quickly recap the characterstics of the stackless C++20 coroutines. [B] As the coroutine frames are allocated on the heap by default, each invocation of a coroutine has to pay the price of a heap allocation. Custom allocators are supported, but need to be managed explicitly. HALO (meaning heap allocation elision optimization) is allowed and in theory often possible, but in practice regularly fails (The standard doesn't mandate HALO, older versions of clang used to elide simple generators, newer versions don't, clang has custom attributes to enforce HALO, but those only work for task-like async coroutines that co_await each other). [B] Additionally, coroutines are type-erased, meaning that for example every generator that yields ints has the type generator of int, independent of its concrete body. Because of this, all access to the coroutine has to go through opaque function pointers which greatly limits the optimizer. [B]In practice, yielding a single int from a generator costs roughly as much as a function call into a different translation unit, even if the generator body is completely visible to the compiler. While this design has many advantages for task-style asynchronous coroutines, it currently hinders the usage of generators as a convenient way to implement tight-loop input ranges. [B] Our stackful design will aim (and achieve!) to lift those limitations.

stackful_coroutine_handle


template <typename CoroFrame>
struct stackful_coroutine_handle {
  CoroFrame frame;  // stored BY VALUE — no heap!

  explicit stackful_coroutine_handle(CoroFrame&& f) noexcept
      : frame(std::move(f)) {}

  // Direct calls — no function pointer indirection
  void resume() { frame.doStep(); }
  void destroy() { frame.destroy(); }
  bool done() const { return frame.done(); }
  auto& promise() { return frame.promise(); }
};
  • Frame stored by value → stack or in registers
  • Direct member calls → fully inlineable
We start again with a custom coroutine handle, this time a stackful coroutine handle that is templated on the coroutine frame. [B] The handle stores the coroutine frame as a member, meaning that it can be stored on the stack or even in registers if simple enough. This is a great difference to stackless C++20 coroutines, where the handle is a very light and type-erased wrapper around a coroutine. [B] As the handle knows the type of the frame, functions like resume and destroy no longer go through function pointers, but become direct member function calls, which can be easily inlined by the optimizer.

stackful_coro_crtp


template <typename Derived, typename PromiseType>
struct stackful_coro_crtp {
  PromiseType promise_;   // no HandleFrame with function pointers!
  size_t suspendIdx_ = 0;

  // ...: doStep(Impl), destroySuspendedCoro etc., identical to stackless.

  // Return the strongly-typed frame by value.
  template <typename... Args>
  static Derived ramp(Args&&... args) {
    Derived frame{std::forward<Args>(args)...};
    // ... await initial suspend (same as for stackless)
    return frame;
  }
};

Generator class: Changes required

  • Templated on coroutine handle, which is templated on coroutine frame
  • Constructible from coroutine frame (can't use promise_type::get_return_object), because promise is not templated on frame.
  • Copy/move of a generator will copy/move the frame
  • Diff to stackless generator can be encapsulated in a templated policy.

Unified Generator Policy


// Same generator template, different policies:

template <typename T>
struct HeapGeneratorPolicy {
  using handle_type = stackless_coroutine_handle<promise_type>;
  static constexpr bool nullable = true;
};

template <typename T, typename CoroFrame>
struct InlineGeneratorPolicy {
  using handle_type = stackful_coroutine_handle<CoroFrame>;
  static constexpr bool nullable = false;
};

// Aliases:
template <typename T>
using stackless_generator = unified_generator<T, HeapGeneratorPolicy<T>>;

template <typename T, typename CoroFrame>
using stackful_generator = unified_generator<T, InlineGeneratorPolicy<T, CoroFrame>>;

iota — Same Frame, Two Policies


template <bool stackless = true>
auto iota_unified(int start, int end) {
  using generator = std::conditional_t<stackless,
      heap_generator<int>::promise_type,
      detail::unified_generator_promise<int>>;

  struct CoroFrame
      : FrameCRTP<stackless, CoroFrame, promise_type, true> {
    // ... same doStepImpl, same destroySuspendedCoro ...
  };

  if constexpr (stackless) {
    return stackless_generator{CoroFrame::ramp(start, end);
  } else {
    return stackful_generator<int, CoroFrame>{
        stackful_coroutine_handle<CoroFrame>{
            CoroFrame::ramp(start, end)}};  // inline_gen
  }
}

Discussion

Stackful vs stackless tradeoffs?

Stackless (heap)

  • Type erasure → polymorphism
  • Symmetric transfer
  • Heap allocation

Stackful (inline)

  • Zero overhead → full inlining
  • No symmetric transfer
  • Type must be known at call site

LibTooling Rewriter

  • The transformation is mechanical — it can be automated
  • Clang LibTooling: parse C++20 coroutine → emit C++17 frame
  • Handles: state machine generation, local variable hoisting, cleanup tables

Limitations

  • Nested co_await in expressions (evaluation order)
  • One lifetime edge case with reference temporaries
  • Complex control flow (e.g., switch with fallthrough + co_yield)

Why Rewrite Language Features?

  • C++17 portability — use coroutine patterns where C++20 isn't available
  • Custom infrastructure — swap out heap for inline, add instrumentation
  • Separation of concerns — the coroutine body is what, the infrastructure is how
  • Understanding — seeing the lowered form makes the protocol concrete

What We Covered

  • Infrastructure: HandleFrame, coroutine_handle, CRTP base, coro_storage, macros
  • 4 Examples:

    • (b) Local variables — coro_storage lifetime management
    • (c) Task — symmetric transfer via trampoline
    • (d) Exceptions — try-block dispatch, catch-clause rewriting
  • Stackful alternative: zero-overhead inline coroutines

Subtleties: frame layout, local lifetimes, exception dispatch, symmetric transfer
